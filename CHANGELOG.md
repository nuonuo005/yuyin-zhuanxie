# 更新日志

本文件记录面向使用者的重要变化。日期采用北京时间。

## v1.3.0 - 2026-07-16

这是一次以稳定性、安全性、内存控制和部署可靠性为重点的更新，不改变原有界面和核心使用方式。

### 修复

- 修复长时间录音把全部音频一直保存在内存中的问题。
- 修复录音临时 WAV 在转写完成后没有自动删除的问题。
- 修复连续触发热键可能造成重复转写、重复 AI 请求或提示词串用的问题。
- 修复同时启动多个实例后重复注册全局热键的问题。
- 修复剪贴板包含图片或文件时，文字读取可能因 Tk 环境异常而报错的问题。
- 修复剪贴板写入失败后仍继续自动粘贴、界面却提示成功的问题。
- 修复命令行模式没有正确处理剪贴板失败和历史数量上限的问题。
- 修复“万一”“千万”等普通中文词语可能被错误转换为数字的问题。
- 修复配置文件写到一半时可能损坏的问题；读取失败时会尝试备份。
- 修复 AI 返回内容被截断时仍把不完整文本当作成功结果的问题。
- 修复 DeepSeek 超时、无权限、请求过多、连接失败等错误提示不清楚的问题。
- 修复安装、打包脚本忽略外部命令退出码，实际失败后仍可能提示完成的问题。
- 修复 Windows PowerShell 5.1 对脚本中文编码解析不稳定的问题。
- 修复 EXE 模式下项目目录仍按源码位置计算的问题。

### 安全

- API Key 改用 Windows DPAPI 加密保存，不再以明文写入 `config.json`。
- 配置保存采用临时文件验证、原子替换和备份副本。
- AI Base URL 只接受 `http` 或 `https`，保存前检查模型名和地址。
- `.gitignore` 增加配置备份、临时配置、独立构建环境和 spec 文件。

### 性能与内存

- 录音过程中直接写入临时 WAV，不再维护无限增长的内存帧列表。
- 增加最长录音时间，默认 600 秒。
- 历史记录默认最多保留 5000 条。
- 保留 ASR/VAD 模型常驻，确保热键转写仍能快速响应。
- 标点模型可根据 AI 润色设置按需加载。
- 绕开 `funasr-onnx` 对无关 SenseVoice 模块的 PyTorch 导入。
- 移除项目未使用的完整版 FunASR、PyTorch、Torchaudio 和 Transformers。

### AI 接口

- DeepSeek 默认模型更新为 `deepseek-v4-flash`。
- 支持 Base URL 和完整 `/chat/completions` 地址。
- 增加连接超时、读取超时、401、429、非 JSON 返回和空内容处理。
- AI 失败时保留本地规则整理结果，不让整次转写完全丢失。

### 部署与打包

- `install.ps1` 可重复运行，不会每次重建虚拟环境。
- 模型命令会优先复制本地缓存，缺失时从 ModelScope 下载。
- `package.ps1` 使用独立 `.build-venv`，避免污染日常运行环境。
- PyInstaller 显式包含项目使用的 FunASR ONNX 模块，并排除无关大模型模块。
- 分发时必须复制完整 `dist\YuyanZhuanxie` 文件夹；模型和个人配置不随包分发。

### 验证结果

- 10 项自动测试全部通过。
- `pip check` 通过，没有断裂依赖。
- 三个本地模型检查通过。
- 使用真实 8.4 秒录音完成本地转写，结果非空。
- 剪贴板写入、读回和恢复测试通过。
- 单实例保护测试通过。
- 源码 GUI 和最终 EXE 均完成隐藏启动存活测试。
- Windows EXE 构建成功。
